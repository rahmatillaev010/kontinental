# Континенталь — портал гильдии Free Fire

Готовый сайт-портал для гильдии «Континенталь»: главная страница, история, лидер, участники, руководство, галерея, правила, анкета, админ-панель и архив.

Сайт работает в двух режимах:

- без Supabase: откроется с демо-данными;
- с Supabase: анкеты, роли и участники будут храниться в базе.

## 1. Установить Node.js

1. Открой сайт [nodejs.org](https://nodejs.org/).
2. Скачай версию LTS.
3. Установи как обычную программу.
4. Перезагрузи компьютер, если установщик попросит.

## 2. Открыть проект

Открой папку проекта:

```text
C:\Users\abdus\Documents\Codex\2026-07-01\codex-1-free-fire-id-2
```

## 3. Установить зависимости

Открой терминал в папке проекта и выполни:

```powershell
npm install
```

Если PowerShell пишет, что `npm.ps1` заблокирован, используй Windows-вариант:

```powershell
npm.cmd install
```

## 4. Создать Supabase

1. Зайди на [supabase.com](https://supabase.com/).
2. Создай аккаунт.
3. Нажми New project.
4. Придумай название, например `kontinental`.
5. Сохрани пароль базы в надёжном месте.
6. Дождись создания проекта.

## 5. Вставить ключи Supabase

1. В Supabase открой Project Settings.
2. Слева нажми `API Keys`.
3. Скопируй `Project URL`.
4. Скопируй публичный ключ `anon` или `publishable key`.
5. Не копируй `service_role` или `secret key` — это секретный ключ администратора.
6. В проекте создай файл `.env.local`.
7. Заполни его так:

```text
NEXT_PUBLIC_SUPABASE_URL=https://your-project.supabase.co
NEXT_PUBLIC_SUPABASE_ANON_KEY=your-public-anon-key
NEXT_PUBLIC_MEMBERS_ACCESS_CODE=kontinental
```

Пример уже лежит в `.env.local.example`.

`NEXT_PUBLIC_MEMBERS_ACCESS_CODE` — это код для раздела «Участники». По умолчанию стоит `kontinental`. Можешь заменить на свой код, например `lider2026`.

## 6. Создать таблицы

1. В Supabase открой SQL Editor.
2. Открой файл `supabase/schema.sql` из проекта.
3. Скопируй весь SQL.
4. Вставь в SQL Editor.
5. Нажми Run.

Этот SQL создаёт таблицы `members`, `applications`, `roles`, `site_content`, `admin_users`, политики безопасности RLS, хранилище фото и демо-данные.

## 7. Создать администратора

1. В Supabase открой Authentication.
2. Нажми Add user.
3. Укажи email и пароль.
4. Открой созданного пользователя и скопируй User UID.
5. В SQL Editor выполни:

```sql
insert into public.admin_users (user_id)
values ('ВСТАВЬ_USER_UID_СЮДА');
```

После этого этот пользователь сможет зайти в админ-панель.

## 8. Запустить сайт на компьютере

```powershell
npm run dev
```

Если PowerShell снова ругается на `npm.ps1`, запусти так:

```powershell
npm.cmd run dev
```

Открой в браузере:

```text
http://localhost:3000
```

Чтобы открыть сайт на телефоне, компьютер и телефон должны быть в одной Wi-Fi сети или на одной точке доступа. На телефоне не открывай `localhost`, открой IP компьютера:

```text
http://IP_КОМПЬЮТЕРА:3000
```

Пример:

```text
http://172.20.10.2:3000
```

## 9. Загрузить проект на GitHub

1. Создай новый репозиторий на GitHub.
2. В терминале в папке проекта выполни:

```powershell
git init
git add .
git commit -m "Create Kontinental guild portal"
git branch -M main
git remote add origin https://github.com/USER/REPOSITORY.git
git push -u origin main
```

Замени `USER` и `REPOSITORY` на свои данные.

## 10. Опубликовать сайт на Vercel

1. Открой [vercel.com](https://vercel.com/).
2. Войди через GitHub.
3. Нажми Add New Project.
4. Выбери репозиторий сайта.
5. В разделе Environment Variables добавь:

```text
NEXT_PUBLIC_SUPABASE_URL
NEXT_PUBLIC_SUPABASE_ANON_KEY
```

6. Нажми Deploy.

## 11. Открыть сайт по ссылке

После публикации Vercel покажет ссылку вида:

```text
https://kontinental.vercel.app
```

Эту ссылку можно отправлять участникам.

## 12. Зайти в админ-панель

1. Открой `/admin` на сайте.
2. Введи email и пароль администратора из Supabase Auth.
3. Если пользователь есть в `admin_users`, откроется панель управления.

## 13. Одобрять анкеты

1. Открой вкладку «Анкеты».
2. Нажми на новую анкету.
3. Проверь и исправь данные при необходимости.
4. Выбери должность.
5. Нажми «Опубликовать».

После этого анкета станет одобренной, а игрок появится в участниках.

## 14. Менять данные участников

1. Открой вкладку «Участники».
2. Выбери участника.
3. Измени поля.
4. Нажми «Сохранить».

Там же можно отправить участника в архив или удалить.

## 15. Менять должности

1. Открой вкладку «Руководство».
2. Выбери должность для редактирования.
3. Измени название, описание, стиль или порядок.
4. Нажми «Сохранить должность».

Можно добавлять новые должности, удалять старые и менять порядок стрелками.

## 16. Менять тексты сайта

1. Открой `/admin`.
2. Войди как администратор.
3. Открой вкладку «Тексты».
4. Выбери нужный блок, например `home_welcome`, `home_guild_info` или `history_intro`.
5. Измени заголовок, текст или ссылку.
6. Нажми «Сохранить текст».

Так можно менять тексты на главной странице и в истории без правки кода.

## Важная логика безопасности

- Анкета создаётся со статусом `pending`.
- Обычный посетитель не может читать анкеты.
- Обычный посетитель не может менять участников и должности.
- Публично видны только участники и должности.
- Раздел «Участники» дополнительно закрыт кодом доступа из `.env.local`.
- Тексты сайта хранятся в `site_content` и меняются только администратором.
- Админ-панель работает только после входа через Supabase Auth.
- Право администратора выдаётся через таблицу `admin_users`.
- Фото загружаются в Supabase Storage в bucket `member-photos`.
